# Locality and Irrelevance

Goal:

- capture non-interference algebra at coordinate/set level.

GameTheory-side patterns:

- update at irrelevant coordinate does not change evaluator,
- finite batches of updates preserve result.

Math-side target:

- `Ignores` closure algebra,
- replacement/update commutation and finite-set invariance.

Equivalent names:

- `Ignores` (chosen):
  - structure: function invariant under changing one coordinate.
  - equivalent names: non-interference, frame-invariance, coordinate-insensitivity.
- `Locality` (chosen):
  - structure: value depends only on a limited dependency footprint.
  - equivalent names: dependency sparsity, support locality, footprint restriction.

External-field fit:

- frame rules in program logic, sparse dependency analysis.

Candidate artifacts:

- closure under composition/product/if-then-else,
- update batch invariance over finite supports,
- commutation and idempotence normalizers.
